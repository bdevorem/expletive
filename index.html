xpl3t!v3

Read the Docs


xpl3t!v3 is an especially esoteric programming language based on Emil Leon Post's tag system from 1943. The underlying data structure is a 2-tag queue, where each time a symbol is read, one more gets popped and an unbounded amount gets pushed. Read the docs to find out how to program in xpl3t!v3.

1. Language Specs


Alphabet


The first step in creating an xpl3t!v3 program is to define an alphabet. This specifies the scope of a program and also ensures that the correct production rules are being performed. If a rule is defined for a symbol that is not in the alphabet, you're lucky and nothing will happen- but this could be the cause of some bugs in your program.
One thing to note: whitespace in xpl3t!v3 is extremely important, due to it's esotericism. In xpl3t!v3, you can define your symbols to be anything you want- even symbols consisting of more than one character. You read that right, you can have a symbol that's more than one character in length. Therefore, pay attention to your spacing. Notice in alphabet below that the symbols a, b, and ab are all defined. To define an alphabet consisting of only the 26 letters of the English alphabet, and the symbol ab:
@|?h ab a b c d e f g h i j k l m n o p q r s t u v w x y z

Halt Symbols


Unless otherwise declared, an xpl3t!v3 program will halt once the queue reaches a length of less than two symbols. If the programmer has other ideas in mind, they can declare a halting symbol in which the program will exit once read. Only one halting symbol can be declared, so the most recently-declared will take priority. An example of a halting symbol, 'H':
h@|+ H

Production Rules


Once the alphabet and halting rules for the program are defined, we can move on to the fun part: how to pop/push! The production rules are the heart of an xpl3t!v3 program: they define how symbols are pushed onto the queue. They do not control how symbols are read or popped, however- the head of the queue will always be read, the top two symbols will always be popped, and the symbols to be pushed will always go on the back of the queue. Additionally, if a rule is not declared for an alphabet symbol, the interpreter will move on as if the mapping had been to the empty string, epsilon. An example of a production rule for mapping an 'a' to the symbols 'b', 'c', and 'd':
r#|3 a b c d
Note that this is different than mapping 'a' to the single symbol 'bcd': r#|3 a bcd
Have we mentioned whitespace is important?

Program Title


If the programmer so desires, a program title can be declared. If so, this string will be printed to the user before operations are performed on the initial configuration. To add a title, simply do the following:
+|+13 This is a Title

Initial Configuration


One more requirement for an xpl3t!v3 program is an initial configuration. The initial configuration is the string of symbols that the queue is initialized to, and the symbols that start the production rules. This can be entered in two ways. If the programmer wants their program to be a generalized set of rules, they can enter the initial configuration from the command line as a flag during interpretation using the following:
-i abcde or --initial abcde
However, if the programmer decides to enter their configuration using this method, then their symbols must only be single characters in length. The reason for this is to avoid confusion with other command line arguments and flags. Therefore, the above configuration would not be the single symbol abcde, it would be the list of single-character symbols a, b, c, d, e. If the user has a very specialized program, they can enter the intial configuration from within the program like so:
n?#+ a b c d e
Note that this is different from:
n?#+ abcde

Comments


What's a good program without some comments? In xpl3t!v3, comments are super easy, but can only be inserted in a line of their own
* This is a comment

Verbose


If you want to see the configuration of the tape at every tape, you can pass a flag during interpretation as follows:
-v or --verbose

3. Usage


Download


xpl3t!v3 programs use an interpreter, written in Python 2 (so make sure you have that installed!) To use the language, you must download the latest release of the interpreter here.

Run


After downloading the interpreter, you can either run the xpl3t!v3 shell or write a program and pass it to the interpreter. To start the shell, navigate to the src/ directory, and type ./xpl.py. To pass in a program written in xpl3t!ve, run this from the command line (where your program is test.xpl, your initial configuration is testing, and you want it displayed in verbose mode):
./xpl.py test.xpl -i testing -v
