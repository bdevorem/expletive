xpl3t!v3

Read the Docs


xpl3t!v3 is an especially esoteric programming language based on Emil Leon Post's tag system from 1943. The underlying data structure is a 2-tag queue, where each time a symbol is read, one more gets popped and an unbounded amount gets pushed. Read the docs to find out how to program in xpl3t!v3.

1. Language Specs


Alphabet


The first step in creating an xpl3t!v3 program is to define an alphabet. This specifies the scope of a program and also ensures that the correct production rules are being performed. If a rule is defined for a symbol that is not in the alphabet, you're lucky and nothing will happen- but this could be the cause of some bugs in your program. To define an alphabet consisting of only the 26 letters of the English alphabet:
@|?h {a b c d e f g h i j k l m n o p q r s t u v w x y z}

Halt Symbols


Unless otherwise declared, an xpl3t!v3 program will halt once the queue reaches a length of less than two characters. If the programmer has other ideas in mind, they can declare a halting symbol in which the program will exit once read. An example of a halting symbol, 'H':
h@|+ {H}

Production Rules


Once the alphabet and halting rules for the program are defined, we can move on to the fun part: how to pop/push! The production rules are the heart of an xpl3t!v3 program: they define how symbols are pushed onto the queue. They do not control how symbols are read or popped, however- the head of the queue will always be read, the top two symbols will always be popped, and the symbols to be pushed will always go on the back of the queue. Additionally, if a rule is not declared for an alphabet symbol, the interpreter will move on as if the mapping had been to the empty string, epsilon. An example of a production rule for mapping an 'a' to 'bcd':
r#|3 {a} {bcd}

Program Title


If the programmer so desires, a program title can be declared. If so, this string will be printed to the user before operations are performed on the initial configuration. To add a title, simply do the following:
+|+13 {This is a Title}

Initial Configuration


One more requirement for an xpl3t!v3 program is an initial configuration. The initial configuration is the string of symbols that the queue is initialized to, and the symbols that start the production rules. This can be entered in two ways. If the programmer wants their program to be a generalized set of rules, they can enter the initial configuration from the command line as a flag during interpretation using the following:
-i abcde or --initial abcde
If the user has a very specialized program, they can enter the intial configuration from within the program like so:
n?#+ {abcde}

Comments


What's a good program without some comments? In xpl3t!v3, comments are super easy, and can be inserted both inline and wherever:
* This is a comment

Verbose


If you want to see the configuration of the tape at every tape, you can pass a flag during interpretation as follows:
-v or --verbose

3. Usage


Download


xpl3t!v3 programs use an interpreter, written in Python 2 (so make sure you have that installed!) To download this interpreter and enable execution, run the following from the command line:
curl https://raw.githubusercontent.com/bdevorem/expletive/master/xpl.py > xpl.py
chmod +x xpl.py

Alternatively, you can download the latest release here.

Run


After downloading the interpreter and writing your program, run from the command line (where your program is test.py, your initial configuration is testing, and you want it displayed in verbose mode):
./xpl.py test.xpl -i testing -v
